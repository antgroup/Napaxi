# 贡献者许可协议

Napaxi 要求外部贡献在合并前完成 Contributor License Agreement（CLA，贡献者
许可协议）。

CLA 用于确认贡献者有权提交相关贡献，并向蚂蚁集团及下游接收方授予在 Napaxi
项目中使用、修改、分发和继续构建这些贡献所需的权利。

## 应该签署哪一份 CLA？

- **个人 CLA**：适用于以个人身份提交贡献的贡献者。
- **公司 CLA**：适用于由公司或其它法律实体拥有或控制的贡献，包括雇佣关系
  或公司资助工作中产生的贡献。

蚂蚁集团官方 CLA 模板：

- [个人贡献者许可协议](docs/legal/cla/Ant_Group_Open_Source_Individual_CLA_English_Chinese_202308.docx)
- [公司贡献者许可协议](docs/legal/cla/Ant_Group_Open_Source_Corporate_CLA_English_Chinese_202308.docx)

上述文件是本项目使用的官方中英文 CLA 表单。请不要在 PR 中修改 CLA 正文。

## Pull Request 要求

PR 合并前需要满足：

1. 贡献者完成适用的 CLA。
2. PR 通过配置好的 CLA 检查；在自动化检查完成前，maintainer 可以人工确认
   已签署 CLA。
3. 如果贡献代表雇主或其它组织提交，maintainer 可以要求提供公司 CLA，并同时
   确认个人贡献者授权情况。

仓库内文档只说明 CLA 要求；真正的自动阻断还依赖代码托管平台配置。维护者应在
大规模接收外部贡献前配置 CLA bot 或 required pull-request check。
