# Contributor License Agreement

Napaxi requires a Contributor License Agreement (CLA) before external
contributions can be merged.

The CLA clarifies that contributors have the right to submit their
contributions and grant Ant Group and downstream recipients the rights needed to
use, modify, distribute, and build upon those contributions as part of Napaxi.

## Which CLA Should I Sign?

- **Individual CLA**: use this when you contribute as an individual.
- **Corporate CLA**: use this when a company or other legal entity owns or
  controls the contribution, including contributions made as part of employment
  or company-sponsored work.

Official Ant Group CLA templates:

- [Individual Contributor License Agreement](docs/legal/cla/Ant_Group_Open_Source_Individual_CLA_English_Chinese_202308.docx)
- [Corporate Contributor License Agreement](docs/legal/cla/Ant_Group_Open_Source_Corporate_CLA_English_Chinese_202308.docx)

These files are the official English/Chinese CLA forms for this project. Do not
edit the CLA text in a pull request.

## Pull Request Requirement

Before a pull request can be merged:

1. The contributor must complete the applicable CLA.
2. The pull request must pass the configured CLA check, or a maintainer must
   manually verify the signed CLA while automated checking is being set up.
3. If the contribution is made on behalf of an employer or other organization,
   maintainers may require the Corporate CLA in addition to the individual
   contributor's confirmation.

The repository documents the CLA requirement here, but enforcement also depends
on the code hosting platform. Maintainers should configure a CLA bot or required
pull-request check before accepting external contributions at scale.
